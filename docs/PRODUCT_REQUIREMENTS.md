# PRODUCT_REQUIREMENTS.md

## Functional requirements
- admin створює або підтверджує користувачів
- користувач змінює тимчасовий пароль при першому вході
- користувач редагує профіль
- користувач додає моніторинг маршруту з періодом дат
- система щогодини опитує провайдерів
- усі знайдені варіанти зберігаються
- при зміні ціни відправляється сповіщення
- підписка живе до cancel або expiry

## Non-functional requirements
- розширюваність по провайдерах
- простота підтримки в Codex/PyCharm workflow
- відокремлення UI, domain logic і integrations
